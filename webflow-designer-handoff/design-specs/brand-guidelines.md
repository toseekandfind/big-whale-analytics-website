## üé® Color Palette

### Primary Colors
- **Primary Blue:** `#1E40AF` (Used for main CTAs, headers, primary elements)
- **Secondary Blue:** `#3B82F6` (Used for secondary elements, links, accents)
- **Accent Orange:** `#F97316` (Used for highlights, important CTAs, alerts)

### Neutral Colors
- **Dark Gray:** `#1F2937` (Main text, headers)
- **Medium Gray:** `#6B7280` (Secondary text, captions)
- **Light Gray:** `#F3F4F6` (Backgrounds, sections)
- **White:** `#FFFFFF` (Primary backgrounds)

### Usage Guidelines
- Use Primary Blue for main CTAs and primary navigation
- Use Secondary Blue for links and secondary buttons
- Use Accent Orange sparingly for highlights and important actions
- Use Dark Gray for all body text and headers
- Use Light Gray for section backgrounds and cards

## üìù Typography

### Font Stack
- **Primary Font:** Inter (Google Fonts)
  - Weights: 400 (Regular), 500 (Medium), 600 (SemiBold), 700 (Bold)
  - Used for: All body text, headers, navigation

- **Code Font:** JetBrains Mono (Google Fonts)
  - Weights: 400 (Regular), 500 (Medium)
  - Used for: Code snippets, technical content, data examples

### Typography Scale
- **H1:** 48px/60px (Inter Bold)
- **H2:** 36px/44px (Inter SemiBold)
- **H3:** 24px/32px (Inter Medium)
- **H4:** 20px/28px (Inter Medium)
- **Body:** 16px/24px (Inter Regular)
- **Small:** 14px/20px (Inter Regular)
- **Caption:** 12px/16px (Inter Regular)

## üéØ Visual Elements

- **Primary Logo:** Big Whale Analytics text with whale icon
- **Logo Colors:** Primary Blue on white, or white on Primary Blue
- **Minimum Size:** 120px width for web use

- **Color:** Primary Blue or Medium Gray

- **Style:** Clean, professional, modern
- **Color Treatment:** Slightly desaturated, professional tone
- **Subjects:** Business professionals, modern office environments, technology
- **Avoid:** Stock photos that look overly staged or generic

## üì± Responsive Design

### Breakpoints
- **Mobile:** 320px - 767px
- **Tablet:** 768px - 1023px
- **Desktop:** 1024px+

### Mobile Considerations
- Stack elements vertically
- Increase touch targets to 44px minimum
- Reduce font sizes by 1-2px
- Use full-width containers
- Simplify navigation to hamburger menu

## üéØ Animation Guidelines

### Transitions
- **Duration:** 0.2s - 0.3s
- **Easing:** Ease-out
- **Properties:** Opacity, transform, color

### Hover Effects
- **Buttons:** Slight scale (1.02) and color change
- **Cards:** Subtle lift and shadow increase
- **Links:** Color change and underline

### Page Transitions
- **Fade In:** 0.3s ease-out
- **Stagger:** 0.1s delay between elements

## üö´ What to Avoid

- Don't use more than 3 colors in a single section
- Don't use fonts other than Inter and JetBrains Mono
- Don't use gradients unless specifically requested
- Don't use heavy shadows or 3D effects
- Don't use bright, saturated colors
- Don't use decorative fonts or script styles

## ‚úÖ Brand Voice

### Tone
- **Professional:** Expert-level knowledge without being condescending
- **Confident:** Clear, direct communication
- **Helpful:** Focus on solving problems, not just selling
- **Modern:** Up-to-date with current data stack trends

### Messaging Principles
- Focus on business outcomes, not just technical features
- Use concrete examples and case studies
- Emphasize partnership and long-term success
- Avoid jargon unless explaining technical concepts

---

**These guidelines ensure consistency across all brand touchpoints and help create a professional, trustworthy impression that converts visitors into clients.**
